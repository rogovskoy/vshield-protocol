VShield - это экспериментальный прокси-протокол нового поколения, разработанный для обхода глубокого анализа трафика (DPI) и устойчивый к атакам с использованием квантовых компьютеров. Проект вдохновлен VLESS, но полностью переосмысливает подход к маскировке и безопасности.

## Ключевые особенности

- Пост-квантовая криптография - гибридное рукопожатие X25519 + Kyber768 (ML-KEM). Даже если квантовый компьютер сломает классическую криптографию, Kyber останется надежным.
- Умная обфускация - трафик не шифруется "в лоб", а маскируется под реальные протоколы: WebRTC (видеозвонки) и HTTP/3 (QUIC). DPI видит знакомые паттерны и пропускает.
- Адаптивность - если соединение начинает "зажиматься", протокол автоматически меняет профиль трафика прямо во время сессии без разрыва соединения.
- Легковесность - только XChaCha20-Poly1305 для данных, никакого лишнего шифрования на уровне сессии.
- Мультиплексирование - поддержка множества потоков через одно UDP-соединение (собственная реализация поверх UDP).

## Как это работает

```
Клиент -> [Обфускация: WebRTC/QUIC] -> [Гибридное шифрование] -> Сервер -> Интернет
         └────────────── X25519 + Kyber768 ────────────────┘
```

1. Рукопожатие: клиент и сервер выполняют гибридный обмен ключами (X25519 для совместимости + Kyber768 для квантовой устойчивости).
2. Маскировка: все пакеты упаковываются в RTCP-фреймы (Sender Reports) или QUIC-пакеты, имитируя видеозвонки или обычный веб-трафик.
3. Шифрование: данные шифруются XChaCha20-Poly1305 с уникальным nonce для каждого пакета.
4. Адаптация: при подозрении на блокировку клиент и сервер синхронно переключаются на другой профиль трафика.

## Компоненты проекта (будет дополняться!!)

- protocol/ - ядро протокола (фреймы, потоки, состояния)
- crypto/ - криптография (Kyber, X25519, ChaCha20)
- transport/ - транспортный уровень (UDP, TCP)
- obfs/ - обфускация (WebRTC, QUIC, Noise)
- congestion/ - контроль перегрузки
- session/ - управление сессиями

## Дорожная карта

- Базовая архитектура и спецификация протокола
- Реализация гибридного рукопожатия (X25519+Kyber768)
- WebRTC-обфускатор (RTCP Sender Reports)
- Базовый UDP-транспорт
- Клиент и сервер (echo-mode для тестов)
- QUIC-обфускация (HTTP/3 маскировка)
- Адаптивный переключатель режимов
- Мультиплексирование потоков
- GUI-клиент (опционально)

## Быстрый старт (для разработчиков)

```bash
# Клонирование репозитория
git clone https://github.com/rogovskoy/vshield-protocol
cd vshield-protocol

# Установка зависимостей
make deps

# Запуск тестового сервера
make run-server

# В другом терминале запуск клиента
make run-client
```

## Требования

- Go 1.21 или выше
- (опционально) Rust + Node.js для сборки GUI
